66 - Select row from `depotMatrix`

74 ### Construct initial solution
75 1. Sort `customerMatrix` on ascending demand.
76 2. Create `depotCustomerAllocation` (is a `vector<vector<int>>`). Create `depotCustomerDemand`.
77 3. Loop over `customerMatrix`:
78  - Create `leftoverDemand = customer.getDemand()`.
79  - While `leftoverDemand > 0`
80  - Find smallest distance to depots, say `smallDepot`.
81  - Add customer.
82  - end
83 
84 4. Loop over depot (m depots):
85  - Sort vehicles based on costs.
86  - Create `customerList` and `customerDropOff`.
87  - Loop over sorted vehicle list:
88  - Create `leftoverCapacity = vehicle.getCapacity()`
89  - while (`leftoverCapacity > 0`) do
90  - Take new customer from `depotCustomerAllocation` (first time random).
91  - Add customer to `customerList` and `customerDropOff`.
92  - Update `leftoverCapacity`.
93  - if (`leftoverCapacity >= demand`) then
94  - Delete from `depotCustomerAllocation`.
95  - else update `depotCustomerDemand`.
96  - end
97  - `route = cheapestInsertion(customerList)`
98  - `vehicle.setRoute(route, deliverQuantities)`
99  - `Customers.addRoute(route, deliverQuantities)`

115 ## Perturbation
116 
117 Three destroy operators (remove q customers) + 1 repair operator (reinserts customers).
118 
119 ### randomRemoval(solution) ###
120 1. Select seed customer from customer list. Store in toBeRemoved.
121 2. Find q-1 nearest customers. Store in toBeRemoved.
122 3. removeCustomers(solution, toBeRemoved) (see below)
123 
124 ### removeCustomers(solution, toBeRemoved)
125 1. For each customer in toBeRemoved:
126  - For each vehicle of customer:
127  - Find place customer in route
128  - Remove customer from route (and dropoff)
129  - Update load, length, serviceTime, route cost
130  - Remove depot+vehicle_id from customer
131 2. toBeReinserted = toBeRemoved (for later use)
132 2. Return destroyed solution and toBeReinserted
133 
134 ### costRemoval(solution) ###
135 1. For customers 1,...,n:
136  - computeRemovalGain(solution, customer) (see below)
137  - store result in vector gains, along with customer_id (gains = [[1,gain_1],...,[n,gain_n]])
138 2. Find q largest gains, store corresponding customer_ids in toBeRemoved
139 3. removeCustomers(solution, toBeRemoved)
140 
141 ### computeRemovalGain(solution, customer) ###
142 1. For each vehicle of customer, compute vehicleRemovalGain:
143  - Find place in route
144  - Distance gain: Use distanceMatrix (multiply by alpha)
145  - Feasibility gain (only if route is infeasible in the first place):
146  - Find old and new travel duration (travel time + service time)
147  - Use arithmetic.
148  - vehicleRemovalGain = distance gain + feasibility gain
149  - totalRemovalGain = sum(vehicleRemovalGain)
150  - totalRemovalGain = totalRemovalGain * uniform(a, b) (a = 0.8, b = 1.2)(RNG).
151 2. return totalRemovalGain
152 
153 ### routeRemoval(solution) ###
154 1. Pick random non-empty vehicle.
155 2. Find corresponding route
156 3. Store at random min(q, route.length()) customers in toBeRemoved
157 4. removeCustomers(solution, toBeRemoved)
158 
159 ### reinsert(solution, toBeReinserted) ###
160 1. demand = demand(toBeReinserted)
161 2. While toBeReinserted is not empty
162  - Select random customer from toBeReinserted.
163  - minInsertion = Infinity
164  - depot = -1, vehicle = -1, idx = -1 (initilize insertion location)
165  - For each vehicle
166  - For each insertion option
167  - Find insertionCost:
168  - Additional travel costs
169  - Infeasibility penalties
170  - If (insertion costs < minInsertion)
171  - minInsertion = insertionCost
172  - update: depot, vehicle, and idx
173  - Insert customer at depot, vehicle, idx:
174  - If (insufficient inventory/capacity)
175  - Allocate as much as possible and split demand
176  - Update demand
177  - If (sufficient inventory and capacity)
178  - Allocate all demand
179  - remove customer from toBeReinserted
180 3. Return solution
181 Note: we can alternatively check in advance for sufficient inventory/capacity, and discard the insertion option
182 if insufficient. One problem is that all insertion options may be discarded (a simple workaround is to repeat the
183 search, now allowing for splits).
184 I suggest moving this to the tuning phase (if it turns out that too many splits are generated,
185 this is one place where we can look.)

188 ## Algorithm Structure ##
189 
190 ### Solutions ###
191 solutions: `best_feasible`, `best_solution`, `s`, `s_prime`
192 f(solution) = cost(solution) + penalty(solution)
193 
194 cost(solution) = sum(travelCosts)
195 penalty(solution) = penalty * travelTimeViolation(solution)
196 penalty is a parameter between `iota_min` and `iota_max = iota_min*delta^100` (or delta^10, whatever) multiplied by
197 factor delta if solution is infeasible
198 
199 
200 ### Initialise
201 `penalty = iota_min (global)`
202 `s = initialSolution()`
203 `best_solution = s`
204 
205 if `s` is feasible:
206 - `best_feasible = s`
207 
208 else:
209 - `best_feasible` = empty (objective value: inf)
210 
211 `T = r*f(s) (0 < r < 1)` (Temperature for simulated annealing)
212 
213 `iter == 1`
214 repeat:
215 - `s_prime` = DestroyAndRepair(s)
216  if f(`s_prime`) < (1 + theta)*f(`best_solution`):
217  - for (routine in routineList):
218  - `s_prime` = routine(`s_prime`)
219  if (improvement):
220  - return to first routine (that is, reenter for-loop) #this is rather extensive,
221  (an alternative is to move to the next routine if the current routine does not yield further improvements)
222 
223  `best_solution` = min(`s_prime`, `best_solution`) #compare objectives, use same global penalty parameter
224 
225  if `s_prime` is feasible
226  - `best_feasible` = min(`s_prime`, `best_feasible`) #compare objectives
227 
228  simulatedAnnealing(`s_prime, s`) (simulatedAnnealing: see below)
229  updatePenalty(`s`)
230  `iter++`
231 
232 until `iter == max_iter` or comp_time > max_comp_time
233 
234 
235 ### updatePenalty(solution) ###
236 
237 if(solution is not feasible)
238 - penalty = min(iota_max, iota * delta)
239 
240 else
241 - penalty = max(iota_min, iota / delta)
242 
243 ### simulatedAnnealing(s1, s2) ###
244 
245 if f(s1) <= f(s2):
246 - T = r*f(s1) (0<r<1)
247  return s1
248 
249 else
250 - diff = f(s1) - f(s)
251  generate random variate u~U(0,1)
252  if (u < exp(-diff/T))
253  - return s1 with probability exp(-diff/T)
254  T = max(r*T, T_min) (0 < r <1, T_min is minimum temperature)

257 ### 1-insertion intra route ###
258 minInsertion = 0
259 For each depot
260 - For each vehicle
261  - For each dummy customer in route
262  - For each insertion option in route
263  - Compute insertionCosts (distance + feasibility)
264  if (insertionCosts < minInsertion)
265  - minInsertion = insertionCosts
266  store depot + vehicle + customer + insertion option
267 
268 Execute best option: (change route + deliveries for (depot,vehicle))
269 ### end ###

272 ### 2-opt intra route ###
273 bestOption = 0
274 For each depot
275 - For each vehicle
276  - For i = 1 to route.size()
277  - c1 = route[i - 1]
278  c2 = route[i]
279  For j = i + 1 to route.size()
280  - c3 = route[j]
281  c4 = route[j + 1]
282  saving = distance - distance[c1,c2] - distance[c3,c4] + distance[c1,c3] + distance[c2,c4]
283  if saving < bestOption
284  - bestOption = saving
285  store i, j
286 
287 Execute best option: newRoute = oldRoute[0,i - 1] + revert(oldRoute[i,j]) + oldRoute[j+1, route.size()], change deliveries analagously
